# Changelog

本文档记录 `CodexAppBar` 的主要用户可见变更。

格式参考 Keep a Changelog，时间按 `YYYY-MM-DD` 记录。

## [2026-04-17]

### Added

- 新增独立的 AppKit 菜单栏控制器，使用 `NSStatusItem` 和 `NSPopover` 承载主菜单内容。
- 新增菜单栏图标恢复链路，包括启动恢复、激活恢复、主窗口恢复、切号恢复和周期巡检恢复。
- 新增主窗口“恢复菜单栏图标”操作。
- 新增多组静态回归脚本，覆盖菜单栏持久化、Dock 显示、额度显示、重置提示、订阅截止时间等关键行为。

### Changed

- 账号卡片由“已使用额度”切换为“剩余额度”展示。
- `5h` 与 `7d` 额度统一显示重置时间；当周期尚未启动时显示“额度尚未激活”。
- 当 `5h` 已耗尽但 `7d` 仍可用时，继续保留完整的 `7d` 卡片，同时将 `5h` 卡片显示为 `0%`。
- 账号排序调整为：当前使用中的账号置顶，额度耗尽账号置底。
- 账号头部增加订阅截止时间展示，并放到 `PLUS / TEAM` 后方；快到期时会自动高亮。
- 文档更新为当前实际行为，包括 Dock + 菜单栏双入口、自动恢复、剩余额度与订阅截止信息。

### Fixed

- 修复菜单栏图标经常消失、切号后短时间缺失、恢复按钮无效等稳定性问题。
- 修复在 SwiftUI 视图更新中发布菜单栏状态导致的转圈和卡死问题。
- 修复“重置倒计时长期不变化”误判为已激活额度的边界情况。
- 修复已经开始消耗 `5h` 或 `7d` 的账号仍显示“额度尚未激活”的问题。

## [2026-03-28]

### Added

- 增加低额度自动切号能力，并在无可切换账号时给出通知。
- 增加打开菜单时重新识别当前活跃账号，以及按分钟轮询的自动切号检查。
- 在账号行中增加 `7d` 重置时间展示。

### Changed

- 切号流程简化为无需重启应用的模式。

### Fixed

- 修复自动切号在每次定时器触发时都执行、没有正确受菜单可见性约束的问题。

## [2026-03-27]

### Fixed

- 删除账号时改为使用 `NSAlert` 二次确认，避免原确认交互在当前桌面环境下不稳定。
